359  // Make the derived class a friend. This seems silly, but is actually
360  // necessary, since derived classes can only access non-public members
361  // through their @p this pointer, but not access these members as member
362  // functions of other objects of the type of this base class (i.e. like
363  // <tt>x.f()</tt>, where @p x is an object of the base class, and @p f one
364  // of it's non-public member functions).

366  // Now, in the case of the @p SparseBlockVanka class, we would like to take
